## 1. 项目背景与目标

- 目标：提供“个性化健身训练 + 饮食管理”的一体化移动应用，包含：根据用户年龄/体重/目标/可用时间自动制定训练与饮食计划、实时训练姿态识别与纠正、拍照识别食物并估算热量、体重与打卡进度报表展示。CS353FZ Research Proposal -Grou… G04-Interim Report
- 形式：移动端 App（iOS/Android），前端使用 React Native；后端采用 Spring Boot 提供 REST API，数据存储 MySQL，缓存/会话使用 Redis；模块化/微服务化便于扩展与性能优化。G04-Interim Report

## 3. 核心功能清单与验收标准

1. 账号与安全

- 邮箱/手机号注册登录、验证码/重置密码；JWT 鉴权；密码加密存储。
- 验收：新用户 1 分钟内完成注册登录；重复登录 Token 刷新正常。

1. 用户画像与目标设定

- 采集：年龄/性别/身高/体重/目标（增肌/减脂/塑形）/可用训练时长；可手动修改。CS353FZ Research Proposal -Grou…
- 验收：修改资料后 3 秒内前端同步；计划可随目标调整重新生成。

1. 个性化训练计划

- 自动生成周/日训练计划（时长、动作序列、组次/频次）；可替换动作、难度自适应。CS353FZ Research Proposal -Grou…
- 验收：不同目标与时长给出差异化方案；替换动作后总体强度保持可控。

1. 实时姿态识别与纠错

- 相机取流 > 关键点检测 > 角度判定 > 提示（文本/语音/震动）；离线/弱网可用为宜。G04-Interim Report
- 验收：在常见动作（深蹲/俯卧撑等）错误姿态下可在 0.5–1.0 s 内给出提示。

1. 饮食拍照与热量估算

- 拍照识别主要食物类别，结合份量估算卡路里与三大营养素；可手动修正。G04-Interim Report
- 验收：常见 30 类食物 Top-1 识别准确率≥指定阈值（例如≥80% 作为阶段目标）；人工修正后写回营养账本。

1. 训练与饮食打卡

- 一键打卡、自动统计当日完成度；饮食记录支持扫码/拍照/手录。
- 验收：离线打卡可在网络恢复后自动同步；当日完成度计算准确。

1. 报表与可视化

- 体重变化曲线、训练打卡日历、饮食热量与宏量营养占比饼图/柱状图。CS353FZ Research Proposal -Grou…
- 验收：选择任意时间窗（7/30/90 天）可在 2 s 内完成渲染。

1. 通知与提醒

- 训练/饮食/饮水/休息提醒；消息中心。
- 验收：本地/远程通知到达率与去重策略正常。

## 4. 非功能性指标（NFR）

- 性能：关键 API P95 < 300 ms；峰值 QPS 目标与扩容策略（缓存、异步队列）。G04-Interim Report
- 稳定性：7×24 可用性≥99.5%，错误率 < 0.1%。
- 安全：传输 HTTPS；敏感信息加密；最小权限；日志脱敏；账号风控基础策略。G04-Interim Report
- 兼容：iOS 15+/Android 9+；常见机型适配。
- 可观察性：结构化日志、APM、基础指标（CPU/内存/连接池/GC）与告警。

## 5. 技术栈与架构约束

- 前端：React Native + TypeScript；状态管理（Redux/Zustand 其一）；路由、网络、相机/多媒体库；国际化与可达性方案。G04-Interim Report
- 后端：Spring Boot；JPA/MyBatis（二选一）；Redis；消息队列（可选）；OpenAPI。G04-Interim Report
- AI：姿态识别（MediaPipe Pose/TFLite/ONNX Runtime）、食物识别（CNN 模型）；可端侧或服务端推理，依据延迟与成本权衡。G04-Interim Report
- 架构：前端—应用层—数据层的分层设计，支持模块化/微服务化演进。G04-Interim Report
